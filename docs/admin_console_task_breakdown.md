# 股票量化平台后台任务拆解清单

## 1. 文档说明

### 1.1 文档目标

本文档将后台管理系统拆解为可执行的开发任务，供项目排期、研发分工和联调使用。

拆解维度包括：

- 前端页面任务
- 后端接口任务
- 数据库任务
- 联调任务
- 测试任务

### 1.2 关联文档

- `docs/admin_console_prototype_spec.md`
- `docs/quant_api_detail_spec.md`
- `docs/quant_er_model.md`

---

## 2. 拆解原则

1. 先主链路，后增强功能
2. 先列表页，后详情页
3. 先查询能力，后编辑和审批动作
4. 一个页面任务尽量对应一组接口任务
5. 联调任务必须明确前后端依赖

## 10. 角色分工建议

## 10.1 前端

建议按页面域拆分：

- FE-A：总览、数据、研究
- FE-B：策略、回测验证
- FE-C：交易、风险、账户
- FE-D：运维、治理

## 10.2 后端

建议按业务域拆分：

- BE-A：平台、权限、配置、任务
- BE-B：数据、主数据
- BE-C：研究、策略、回测
- BE-D：交易、风险、账户

## 10.3 测试

建议按链路拆分：

- QA-A：数据与研究
- QA-B：策略与回测
- QA-C：交易与风控
- QA-D：治理与权限

---

## 11. 完成定义

某个后台模块可以认为完成，至少需要满足：

- 页面可访问
- 列表和详情功能可用
- 关键操作可执行
- 权限控制生效
- 审计日志记录完整
- 前后端联调通过
- 测试用例覆盖主链路

---

## 12. 结论

这份任务拆解清单的目标，是让后台系统建设从“知道要做什么”进入“知道谁做、先做什么、依赖什么、怎么联调”的阶段。下一步如果继续推进，最适合做的是：

1. 把 P0 页面画成低保真原型
2. 为 P0 接口生成 OpenAPI 草案
3. 为 P0 表结构生成 DDL 初稿
