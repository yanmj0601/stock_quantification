# 股票量化平台后台页面原型说明书

## 1. 文档说明

### 1.1 文档目标

本文档用于将后台管理系统设计进一步细化为页面原型级说明，明确每个页面的：

- 页面目标
- 适用角色
- 页面布局
- 展示内容
- 关键交互
- 状态与权限
- 对接接口

本文档不直接输出视觉稿，而是输出可用于产品原型、前端实现和后端联调的页面说明。

### 1.2 关联文档

- `docs/current_gap_and_admin_console_design.md`
- `docs/quant_implementation_spec.md`
- `docs/quant_api_detail_spec.md`

### 1.3 设计原则

1. 先服务核心业务链路
2. 一个页面只解决一类主要问题
3. 列表页负责检索和筛选，详情页负责分析和操作
4. 所有关键对象都可跳转到上下游对象
5. 所有关键操作都应带状态反馈和审计留痕

---

## 2. 后台整体结构

## 2.1 一级导航

建议后台一级导航如下：

1. 总览看板
2. 数据中心
3. 研究中心
4. 策略中心
5. 回测验证
6. 交易中心
7. 风险中心
8. 账户中心
9. 运维中心
10. 治理中心

## 2.2 全局布局

建议后台统一采用三段式布局：

- 顶部：全局搜索、环境标识、用户菜单、通知入口
- 左侧：一级导航 + 二级导航
- 主区域：页面内容

全局固定组件：

- 环境标识条
- 全局时间与市场状态组件
- 全局消息通知抽屉
- 全局快捷操作入口

## 2.3 全局交互规范

### 列表页统一能力

- 搜索
- 筛选
- 排序
- 分页
- 导出
- 批量操作

### 详情页统一能力

- 基础信息卡片
- 状态标签
- 时间轴
- 关联对象跳转
- 审计记录入口

### 异步任务统一能力

- 点击动作后返回任务受理状态
- 顶部弹出任务通知
- 页面内支持刷新任务状态
- 任务失败可跳转日志

## 3. 页面原型说明

## 3.1 总览看板

### 页面编码

- `dashboard_overview`

### 页面目标

为平台负责人、策略负责人、风控和运维提供全局运行概览。

### 适用角色

- 管理员
- 投资经理
- 风控负责人
- 运维负责人

### 页面布局

1. 顶部概览卡
2. 中部业务状态区
3. 底部待处理事项区

### 页面模块

#### A. 平台总览卡片

显示：

- 今日运行任务数
- 成功任务数
- 失败任务数
- 今日策略运行数
- 今日风险事件数
- 今日成交订单数
- 待审批数

#### B. 数据健康区

显示：

- 各市场最新数据同步时间
- 数据质量异常数量
- 数据源健康状态

图表：

- 数据域状态矩阵
- 过去 7 天同步成功率

#### C. 策略运行区

显示：

- 今日已运行策略
- 运行失败策略
- 待复核策略
- 最近表现异常策略

#### D. 交易与风险区

显示：

- 今日调仓计划数
- 今日订单数
- 风险拦截数
- 异常订单数

#### E. 待办事项区

显示：

- 待审批事项
- 高优先级告警
- 最近失败任务
- 最近异常账户

### 关键交互

- 点击卡片跳转对应模块列表
- 点击异常项跳转详情页
- 支持按市场切换概览视图

### 对接接口

- `/api/v1/ops/health`
- `/api/v1/ops/tasks`
- `/api/v1/ops/alerts`
- `/api/v1/approvals`
- `/api/v1/reports/strategy-performance`

## 4. 页面状态设计

所有页面至少需要处理以下状态：

- 初始加载
- 加载中
- 空状态
- 成功状态
- 失败状态
- 无权限状态

空状态建议：

- 提供简短说明
- 提供创建入口或刷新入口

失败状态建议：

- 显示错误摘要
- 提供重试按钮
- 提供查看日志入口

- 研究员：数据中心、研究中心、策略中心、回测验证

- 风控：风险中心、审批中心

- 提交订单

- 发起审批
- 审批通过 / 拒绝

- 回滚配置
